## 2. 需求背景与目标

### 2.1 背景

需要提供一个可运行的数据库 IDE 工作台 Demo，用于验证“隐形设计 + 内容即界面”的高信息密度交互方案，并支撑评审、联调与回归。

### 2.2 目标

- 目标 1：完整展示三窗格 IDE 主路径（Schema 树、SQL 编辑区、结果集）。
- 目标 2：覆盖响应式布局（Mobile/Tablet/Desktop）与关键交互（折叠、拖拽、执行查询）。
- 目标 3：验证语义化 Design Tokens、排版、状态反馈和动效参数可落地。

### 2.3 非目标（明确不做）

- 不接入真实数据库与后端查询服务。
- 不实现完整 SQL 语言服务（仅保留演示级能力）。
- 不做跨页面业务流程整合。

## 4. 需求范围

### 4.1 In Scope（本次范围）

1. IDE 三窗格布局与面板分割。
2. Schema Tree 层级导航、选中态与折叠展开。
3. Query Editor 区域、Run Query 按钮状态跃迁。
4. Data Grid 结果展示、Hover 反馈、空态/错误态。
5. 主题变量映射（`--background`、`--card`、`--muted`、`--border`、`--primary` 等）。
6. 断点行为：`<768`、`768-1024`、`>1024`。

### 4.2 Out of Scope（不在本次范围）

1. 真实 SQL 执行与权限控制。
2. 生产级性能优化（超大数据量虚拟滚动等）。
3. 多工作区、多连接管理。

---

## 5. 详细需求清单（按优先级）

| 编号 | 优先级 | 需求描述 | 触发方式 | 期望表现 | 验收标准 |
|---|---|---|---|---|---|
| D-01 | P0 | 提供 Desktop 三窗格 IDE 布局（左树右上下） | 进入页面 | 默认展示完整 IDE 结构，分隔线清晰 | 首屏可见 Tree、Editor、Grid 三块区域 |
| D-02 | P0 | 支持 Run Query 状态跃迁（Active/Loading/Success 或 Error） | 点击 Run 或 `Ctrl/Cmd + Enter` | 按钮缩放、加载态、结果区状态同步切换 | 状态切换无卡死，结果区反馈正确 |
| D-03 | P0 | Data Grid 支持行 Hover 快速反馈与表头粘性 | 鼠标移动/滚动结果区 | 行背景 75ms 内响应，表头固定 | Hover 无明显拖影，表头保持可见 |
| D-04 | P1 | 响应式重构：Mobile 单列、Tablet 折叠侧栏、Desktop 三窗格 | 调整视口宽度 | 布局按断点切换，主要功能可操作 | 三断点下均可完成一次查询演示 |
| D-05 | P1 | Schema Tree 选中态与层级缩进符合规范 | 点击树节点 | 2px 高亮饰条 + 行背景高亮 + 12px 层级缩进 | 树结构层级清晰、选中态稳定 |
| D-06 | P2 | 提供控制面板切换 Empty/Error/Edge 场景 | 控制面板切换 | 编辑区与结果区数据即时切换 | 可复现空结果、异常提示、边界数据 |

## 6. UI 与交互约束

### 6.1 组件选型顺序（强制）

1. 优先 `@repo/ui-shadcn`
2. 智能场景优先 `@repo/ui-ai-elements`
3. 无覆盖时按顺序评估：`@repo/ui-reka` -> `tailwindcss` -> 原生 CSS

### 6.2 交互状态覆盖（至少）

- 默认、Hover、Focus、Active、Disabled
- Loading、Empty、Error
- 边界输入（超长 SQL、空查询、非法查询文本）

### 6.3 可读性要求

- 页面必须可区分：工作台区域、控制区、状态说明区。
- 保持工具型界面的低噪音视觉，不叠加无关装饰。
- 文本与数据对比度满足 WCAG AA。

## 7. 数据与状态策略

### 7.1 数据来源

- 静态 mock + 本地状态机（不连真实后端）

### 7.2 Mock 约束

必须建立 `mock/` 目录并按模块拆分，示例结构：

```txt
apps/studio-web/src/components/playground/database-ide/
├── mock/
│   ├── index.ts
│   ├── schema-tree.mock.ts
│   ├── query-examples.mock.ts
│   ├── query-result.mock.ts
│   ├── query-error.mock.ts
│   └── state-scenarios.ts
```

- 命名语义化，禁止 `mock1/mock2`。
- 场景数据稳定可复现；异常场景需显式可触发。

### 7.3 状态控制方式

- 页面控制面板 + 快捷键触发（`Ctrl/Cmd + Enter`）+ 可选 URL 场景参数

## 11. 风险、依赖与回滚

### 11.1 风险清单

| 风险 | 影响 | 概率 | 应对策略 |
|---|---|---|---|
| 编辑器能力差异（Monaco/CodeMirror）导致表现不一致 | 中 | 中 | 先封装统一交互层，限制 Demo 功能面 |
| 动效过多影响工具感与性能 | 中 | 中 | 仅保留规范中的必要状态动效 |
| Mock 场景覆盖不足导致评审结论失真 | 中 | 中 | 预置 Empty/Error/Edge 固定场景并加控制开关 |

### 11.2 外部依赖

- UI 设计规范：`doc/ui/database-ide.md`
- 组件体系：`@repo/ui-shadcn`

### 11.3 回滚策略
